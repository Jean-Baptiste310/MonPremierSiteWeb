  • SAE 1.01 - Implémentation d'un besoin client

    Description : L'objectif de cette SAE était de créer une liste de jeux codés en C++. Le client/utilisateur devait avoir la possibilité de choisir n'importe quel jeu via une interface. J'ai donc créé un code qui permet de créer un MasterMind et un Juste Prix. Pour ce faire, j'ai tout d'abord, pour le MasterMind, créé des tableaux afin de pouvoir stocker les différentes informations comme la bonne combinaison de couleurs, la combinaison choisie par l'utilisateur... Ensuite, j'ai utilisé des boucles afin de pouvoir donner un nombre de chances à l'utilisateur (ici 10) et pour pouvoir comparer les tableaux corrects et les tableaux proposés par l'utilisateur. Et pour le Juste Prix, on part sur le même principe de boucles et de conditions afin de pouvoir dire à l'utilisateur si le prix donné est plus petit ou plus grand que le bon prix.

  • SAE 1.02 - Comparaison d'approches algorithmiques

    Description : Lors de cette SAE, l'objectif était de réaliser une intelligence artificielle qui joue au morpion. Pour ce faire, il fallait utiliser différentes approches (exemple : simulation de coups, approche experte avec un raisonnement humain ou autre). Il nous fallait également faire des relevés sur les temps d'exécution de nos IA afin de pouvoir les optimiser. Pour ce faire, je suis parti sur une IA qui analyse des fenêtres de cases dans une grille de morpion et qui renvoie un score. Plus celui-ci est élevé, meilleur est le coup. C'est donc une approche qui se base sur de la simulation. J'ai utilisé des boucles, des conditions et des tableaux. Mon programme se divise en 2 fonctions : une qui observe le tableau et l'autre qui l'utilise.

  • SAE 1.03 - Installation d'un poste pour le développement

    Description : Lors de cette SAE, nous devions donner les hardware (composants d’un PC) pour 2 clients. Il nous était également demandé de réaliser des machines virtuelles afin de proposer des PC avec des préinstallations pour les deux clients. Le premier client est une famille de 5 personnes demandant un ordinateur avec un prix maximum de 800 euros. L’utilisation de l’ordinateur est principalement tournée autour de la navigation internet (streaming inclus), de la bureautique et nécessite Windows. La mère de famille et l’adolescent sont occasionnellement gamers et aimeraient, si possible, faire quelques jeux pas trop gourmands. Chaque utilisateur souhaite avoir son profil personnel, mais le pré-adolescent doit avoir un contrôle parental. Enfin, le hardware devait être en anglais et en français. Pour l’étudiant, il fallait un PC avec le budget le plus réduit possible tout en proposant des composants permettant une légère amélioration. Pour finir, il fallait réaliser une VM Ubuntu pour l’étudiant en proposant des scripts d’automatisation.

  • SAE 1.04 - Création d'une base de données

    Description : Lors de ce projet, nous devions créer une base de données pour une agence de voyage « Little Booking » qui désire mettre en place un site web pour permettre à ses clients de réserver des hôtels, appart-hôtels et autres logements, réserver leurs vols ou encore réserver des séjours entiers (vols et hébergements compris). Ce projet nous a demandé de réaliser un dictionnaire de données, un MCD/MLD, le code SQL de celui-ci et enfin des requêtes SQL afin de pouvoir montrer comment se manipulent nos tables. Pour ce faire, nous avons utilisé des outils comme Looping pour la réalisation du MCD et Workbench MySQL pour la réalisation du code et des requêtes (le dictionnaire de données a été réalisé sur un document Word). En plus de tout ça, nous devions également faire un rapport écrit qui donnait les informations sur nos prises de décision.
